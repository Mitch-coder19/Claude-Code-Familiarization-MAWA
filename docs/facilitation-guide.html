1 · What this is & how to run the session

Agentic coding is not autocomplete. Claude Code is a working partner that reads the files in a project, runs commands, writes and edits code across the whole codebase, checks its own work, and iterates until the job is done. You describe the outcome in plain English — Claude figures out the steps.

The shift we want this cohort to feel: you stop typing every line and start delegating multi-step technical work. Instead of "write this one function," you say "automate this onboarding runbook end-to-end, add a safety switch, and test it" — and you review the result.

This session is a familiarization, not a training course. The goal is a single, vivid realization for each person: "I could hand a real task from my week to this." Lead with a live demo, keep each one to a clean before → after, and let people try it themselves.

How to use this playbook

Section 4 has a demo card per function with copy-paste-ready prompts. Pre-stage the companion sample files (Section 7) so every demo starts instantly. Run the 60-minute agenda in Section 5, lean on the facilitation tips in Section 6, and leave this document with attendees afterward.

2 · Ground rule — synthetic data only

⚠ Non-negotiable: synthetic data only

Every demo uses synthetic or non-production data. Make-A-Wish handles eligibility and medical PHI. Never paste real wish-kid, donor, or patient data into a demo — not into a prompt, not into a sample file, not "just to see what happens." Use the provided fake files for every exercise.

If an attendee wants to try their own real task, have them describe it in the abstract or rebuild it with invented data. When in doubt, leave it out.

6 · Facilitation tips

  • Pre-stage the sample files (Section 7) before the room fills, so every demo starts instantly — no fumbling for a file mid-demo.
  • Show one clear before → after per demo. The contrast is the point: the messy input, then the finished result. Resist the urge to show every step.
  • Close by making it personal. Have each person name one real task from their week they'll try next — that's what turns a demo into adoption.

Tip

Keep demos to a single outcome each. If a demo sprawls, you lose the room. The flagship per function is enough — depth beats breadth in 5 minutes.

7 · Sample demo files
